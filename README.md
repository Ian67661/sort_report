+## 1. 排序法介紹
+本報告分析以下五種排序演算法：
+- **冒泡排序 (Bubble Sort)**
+- **選擇排序 (Selection Sort)**
+- **插入排序 (Insertion Sort)**
+- **合併排序 (Merge Sort)**
+- **快速排序 (Quick Sort)**
+
+## 2. 複雜度分析
+| 演算法 | 時間 (平均) | 時間 (最差) | 空間複雜度 |
+| :--- | :--- | :--- | :--- |
+| Bubble Sort | $O(n^2)$ | (n^2)$ | (1)$ |
+| Selection Sort | (n^2)$ | (n^2)$ | (1)$ |
+| Insertion Sort | (n^2)$ | (n^2)$ | (1)$ |
+| Merge Sort | (n \log n)$ | (n \log n)$ | (n)$ |
+| Quick Sort | (n \log n)$ | (n^2)$ | (\log n)$ |
+
+## 3. 實驗設計
+- **開發語言**：Python
+- **測試環境**：Windows 10 / Python 3.x
+- **數據來源**：使用 `random` 模組產生範圍 0-10000 的隨機整數。
+- **數據規模 (n)**：100, 500, 1000, 2000, 5000。
+

+

+3. Merge Sort 與 Quick Sort 在大數據量下表現極其穩定且快速。
+
+## 5. 心得與結論

+
+### 個人觀察：
+- **實用性**：雖然 Bubble Sort 最容易理解，但在實際應用中幾乎不會使用，因為其效率低落。
+- **穩定性**：在需要保持相同數值原始順序的場景下，Merge Sort 是比 Quick Sort 更好的選擇。
+- **空間換取時間**：Merge Sort 雖然快速，但需要額外的 (n)$ 空間，這在記憶體受限的環境需要列入考量。
